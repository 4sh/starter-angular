# Changelog

Toutes les évolutions notables de ce Design System sont documentées dans ce fichier.

Le format suit [Keep a Changelog](https://keepachangelog.com/fr/1.1.0/) et le projet adhère à [Semantic Versioning](https://semver.org/) (voir [VERSIONING.md](./VERSIONING.md)).

> Phase de construction des fondations : tout reste versionné `0.1.0`. Le versioning
> sémantique sera activé une fois la base stabilisée.

## [Unreleased]

### Fixed
- `ui-button` : suppression d'un `NG0951` (« Child query result is required but no value is available ») émis à chaque rendu. L'effet de forwarding des `buttonProps` lisait le `viewChild.required('#host')` dans un `effect()` nu, exécuté pendant `runEffectsInView` avant la résolution de la requête de vue (le `#host` est sous `@if/@else`). Basculé en `afterRenderEffect` (browser-only, vue garantie résolue) avec court-circuit quand `buttonProps` est absent : plus d'erreur dans le `ErrorHandler`, comportement de forwarding inchangé

### Added
- `ui-menu` : menu de **navigation / commandes** headless, statique ou **popup**, inspiré de PrimeNG `p-menu` (+ le repli de `p-panelmenu`). Modèle déclaratif `items` (`UiMenuItem[]`) : **groupes à en-tête** (label + séparateur plat entre sections), **sous-menus imbriqués repliables** — repliables **par défaut au-delà du premier niveau**, surchargeable par item via `toggleable` (forcer un groupe racine repliable / figer un groupe imbriqué ouvert) —, entrées `command` (`{ originalEvent, item }`), navigation **router** (`routerLink`/`queryParams`/`routerLinkActive` stylé par les tokens `-active`) et **URL externes** (`url`/`target`, `rel` sécurisé auto), `disabled`/`visible`/`title`/`ariaLabel`/`styleClass` par item. État d'ouverture **contrôlable** via `expandedKeys` (two-way `model`, objet `{ [id]: boolean }` indexé sur l'`id` des items). Mode **`popup`** : panneau dans un **overlay CDK** ancré à l'élément déclencheur (`toggle(event)`/`show(event)`/`hide()`), retournement auto au-dessus (`autoFlip`), fermeture clic extérieur/`Échap`/activation avec **retour du focus au déclencheur**, entrée animée `slide-down` (système de motion). Repli des sous-menus par la technique **grid-rows** animée (chevron rotatif), coupée par `[motion]="false"` / `prefers-reduced-motion` / `data-motion="off"`. **Clavier roving WAI-ARIA menu** : tab stop unique, `↑`/`↓` en boucle, `Début`/`Fin`, `→`/`←` ouvre/ferme un groupe, `Entrée`/`Espace` active, `Échap` ferme. A11y : `role="menu"`/`menuitem` sur de vrais `<a>`/`<button>`, groupes `role="group"` + `aria-labelledby`, toggles `aria-expanded`+`aria-controls`, contenu replié `inert`, focus ring `--global-high-stroke-focused` (la famille `navigation` n'a pas d'état `focused`). **Templates** `#item` (contenu du menuitem, `$implicit`), `#submenuheader`, slots `#start`/`#end`. Familles de couleur `level` `high`/`low` (items 100 % tokens `navigation.*`), **densités `size` `default`/`small`** (map SCSS `$sizes` — rendu compact pour les menus d'actions « … »). Panneau **sans bordure** (élévation par l'ombre partagée) au **fond opaque `global`** (le token `navigation.*-surface-default` est transparent par design), coque via le **mixin partagé `overlay-panel`**, constantes SCSS locales `$sizes`/`$panel-surface`/`$item-radius`/`$group-indent`
- **Recette partagée `overlay-panel`** pour les panneaux flottants (dropdowns) : groupe de constantes structurelles `$overlay-panel-*` dans `_ui-config.scss` (radius, padding, ombre, épaisseur de trait, offset déclencheur↔panneau — catégorie **global**) + mixin `utils.overlay-panel($surface, $stroke)` (`src/styles/src/utils/_overlay.scss`). La **structure** est partagée entre tous les panneaux flottants (menu, calendrier du datepicker, futurs select/multiselect…), les **couleurs** restent le choix de la famille de tokens de chaque composant (menu → `navigation.*`, forms → `form.*`) — re-styler une famille n'impacte pas les autres ; échappatoire par instance via les custom properties `--ui-overlay-*`
- `ui-nudger` : **incrémenteur numérique** headless (`[−] valeur [+]`) — deux `ui-button` icon-only (`−`/`+`) encadrant la valeur courante, sans champ de saisie (cas « stepper » distinct de `ui-input-number`). Valeur `number` via `ControlValueAccessor` (`BaseControlValueAccessor`) : autonome, `[(ngModel)]`, **Reactive Forms** et **Signal Forms**. Bornes `min`/`max` + pas `step` : le bouton concerné est **désactivé** dès la borne atteinte — les états Figma `Min`/`Max` sont **dérivés** de la valeur, jamais des props (idiome Angular). Tailles `default`/`small` (valeur en typo `title/xl`/`title/lg`), familles de couleur `level` (tokens `actions.*`, transmises aux boutons), `disabled`/`readonly`/`invalid`, icônes et libellés a11y des boutons personnalisables (`decrementIcon`/`incrementIcon`/`decrementLabel`/`incrementLabel`), formateur d'affichage optionnel `formatValue` (unité/symbole), `defaultValue` (valeur de repli / autonome). **A11y** : racine `role="group"` avec nom accessible obligatoire (garde-fou dev sur `ariaLabel`/`ariaLabelledBy`), boutons `<button>` natifs opérables au clavier, valeur en **région live** (`aria-live="polite"`) annoncée à chaque changement, `emitTouch` au `focusout` réel (hors va-et-vient entre les deux boutons). Sortie `valueChange`. Co-localisé, 100 % tokens `form`/`actions`/metrics/typography, `$sizes` en constante SCSS locale. Sans équivalent direct PrimeNG
- `ui-tabs` : onglets **headless** en **API de composition** (`ui-tabs` conteneur + `ui-tab-list` / `ui-tab` + `ui-tab-panels` / `ui-tab-panel`), inspiré de PrimeNG `p-tabs`. Le conteneur pilote l'état actif (`value` two-way `model`), l'`orientation` (`horizontal` / `vertical`), le `scrollable` (zone défilante + navigateurs précédent/suivant `showNavigators` gated sur le débordement), le `lazy` (défaut de groupe, surchargeable par panneau) et le `selectOnFocus` (activation automatique au focus). Chaque onglet est un `<button role="tab">` **natif** (hôte `ui-tab` en `display: contents` → bouton directement dans le `role="tablist"`), avec **navigation clavier roving WAI-ARIA** (`←`/`→` ou `↑`/`↓` selon l'axe en boucle, `Début`/`Fin`, `Entrée`/`Espace` pour activer), **tab stop unique** (roving `tabindex` sur l'onglet actif / premier disponible), `disabled` natif (sauté au clavier), `icon`/`iconPos` optionnels. Structure ARIA complète : `role="tablist"` (`aria-orientation`) → `role="tab"` (`aria-selected` + `aria-controls`) → `role="tabpanel"` (`aria-labelledby` + `tabindex="0"`), panneaux **conservés dans le DOM** mais `hidden` quand inactifs (état préservé) ou rendus paresseusement (template `#content` pour n'initialiser qu'à l'activation). **Indicateur d'onglet actif glissant** mesuré (ResizeObserver + `afterRenderEffect`, offset/longueur du bouton actif) cadencé par le **système de motion** partagé, **personnalisable** via les variables CSS exposées `--ui-tabs-active-bar-color` / `--ui-tabs-active-bar-size` ; panneaux en **fondu** à l'activation (keyframes du système de motion). Mode **menu de navigation** (onglets seuls sans panneaux, contenu fourni par un `router-outlet` via `[(value)]`). Sortie `tabChange`. Coupure de motion par `[motion]="false"`, `data-motion="off"` et `prefers-reduced-motion`. 100 % tokens `navigation` (liste/onglets) + `global` (contenu/trait), constantes SCSS locales `$tab-min-height`/`$bar-thickness`/`$track-thickness`/`$nav-size`
- `ui-link` : **lien textuel inline** headless — un vrai `<a>` destiné au fil du texte (paragraphes) et à la navigation, stylé avec la famille sobre `actions.low.content.*` (light **et** dark). Destination **externe** (`href`) **ou interne** (`routerLink`, directive Angular) ; `external` force `target="_blank"` + `rel="noopener noreferrer"` sûr, `rel`/`target` surchargeables. Icônes optionnelles de chaque côté via **présence d'input** (`iconLeft`/`iconRight`, noms FontAwesome — idiome Angular équivalent aux booléens `Icon Left`/`Icon Right` de Figma), mode **icon-only** déduit (sans `label`, avec garde-fou a11y dev sur `ariaLabel`). Tailles `default`/`small`. **Souligné par défaut** (a11y : la couleur n'est jamais le seul repère d'un lien inline — ajout intentionnel au spec Figma qui n'encode que la couleur), retiré en icon-only. États interactifs (hover/focus/pressed/disabled) **pilotés en CSS par les tokens**, jamais en props ; `disabled` propre à l'ancre (retrait de `href`/`routerLink`, `aria-disabled`, hors tabulation). Sorties `linkClick`/`linkFocus`/`linkBlur` + méthode `focus()`. Co-localisé, 100 % tokens `actions`, constantes SCSS locales `$sizes`/`$focus-ring-width`
- `ui-modal` : fenêtre modale **headless** — surface flottante stylée par tokens rendue au-dessus d'un masque de viewport, pilotée par le modèle **two-way `visible`**. Entrée/sortie via le **système de motion** (`animate.enter`/`animate.leave` + `[uiMotion]` : fondu du masque + `zoom` du dialogue par défaut, preset surchargeable via `motion`/`motionDisabled`). Trois régions composables : `header` (chaîne) ou template `#header`, corps `<ng-content>` (**scroll interne** au-delà de `max-height: 90vh`), pied via template `#footer` (+ `#closeicon`). **Mode modal** (défaut) : masque qui assombrit (`--primitives-black-50`, surchargeable via `--ui-modal-mask-background`) et **bloque le scroll de fond** (compteur partagé + compensation de la gouttière de scrollbar → pas de saut de mise en page) ; `dismissableMask` ferme au clic extérieur. **Mode non-modal** (`modal=false`) : arrière-plan interactif et scrollable (masque traversant), `blockScroll` peut tout de même figer le scroll. **Déplaçable** (`draggable`, poignée d'en-tête, offset via la propriété CSS indépendante `translate` pour ne pas heurter l'animation `zoom`, `keepInViewport`/`minX`/`minY`), **agrandissable** (`maximizable`, plein viewport ↔ restauration, `aria-pressed`), **redimensionnable** (`resizable`, poignée au coin bas-droit façon `textarea` — pointeur uniquement, tailles minimales `minWidth`/`minHeight`, sortie `onResizeEnd`), **9 positions** (`center`/`top`/`bottom`/`left`/`right` + 4 coins), largeurs **par palier** (`breakpoints`, feuille de style runtime scoping `data-modal-uid`), mode **`contained`** (scope à l'ancêtre positionné en `position: absolute` + pas de blocage du scroll — pour embarquer/prévisualiser une modale dans un conteneur), `dialogStyle`/`styleClass`/`maskStyleClass`, empilement `autoZIndex`/`baseZIndex`. **A11y** : `role="dialog"` + `aria-modal`, nom accessible via `aria-labelledby` (titre) sinon `ariaLabel`/`ariaLabelledBy` (garde-fou dev), **piège de focus + capture/restauration** (`@angular/cdk/a11y` `cdkTrapFocus`/`cdkTrapFocusAutoCapture`, `focusTrap`/`focusOnShow`), fermeture `Échap` (`closeOnEscape`+`closable`), boutons d'en-tête `<button>` natifs, respect de `prefers-reduced-motion` / `data-motion="off"`. Sorties `onShow`/`onHide`/`onMaximize`/`onDragEnd`. API inspirée de PrimeNG `p-dialog`. Rendu **en place** (masque `position: fixed`, non téléporté vers `body` — cohérent avec `ui-toast`). 100 % tokens `global`/metrics, constantes SCSS locales `$modal-*`/`$dialog-*`/`$mask-background`
- `ui-segment-control` : contrôle segmenté headless (équivalent d'un *select button*) — choix d'**une** valeur (mode simple) ou de **plusieurs** (`multiple`, le modèle devient un tableau) parmi une courte liste de segments inline. Chaque segment est un `<button>` natif ; le groupe porte la sémantique **WAI-ARIA** (`radiogroup`/`radio` + `aria-checked` en simple avec **navigation aux flèches et sélection au déplacement** — motif radio ; `group`/`aria-pressed` en `multiple`), **roving tabindex** (un seul segment tabbable, flèches ←/→ ou ↑/↓ selon `orientation` + Début/Fin). Options **polymorphes** : primitives, objets `{ value, label?, icon?, disabled?, ariaLabel? }`, ou objets métier via mapping de champs (`optionLabel`/`optionValue`/`optionDisabled`/`optionIcon`) et `dataKey` pour l'égalité des valeurs objet. En mode simple, **indicateur glissant** mesuré (ResizeObserver + `afterRenderEffect`) qui glisse sous le segment actif, cadencé par le **système de motion** (`utils.motion-transition` + garde `motion-reduce`, opt-out `[motion]="false"`, `prefers-reduced-motion` et `data-motion="off"`) ; en mode `multiple`, chaque segment coché peint sa surface. Tailles `default`/`small`, `fluid` (pleine largeur, segments à parts égales), `orientation` `horizontal`/`vertical`, **template de segment** via `[itemTemplate]` (contexte `{ $implicit, option, selected, index }`), `allowEmpty` (autorise le vidage / empêche de vider le dernier en `multiple`), `disabled`/`readonly`/`invalid`. Sorties `selectionChange` et `optionClick`. Valeur via `ControlValueAccessor` (`BaseControlValueAccessor`) : compatible autonome, `[(ngModel)]`, **Reactive Forms**, et **Signal Forms** (`[field]`, via l'interop CVA) ; style **erreur automatique** quand le contrôle est invalide et touché/modifié. Garde-fous a11y en dev (nom accessible du groupe, segment icon-only sans `ariaLabel`). 100 % tokens `actions`/`global` (bordure de piste en `box-shadow` inset → math d'offset exacte), gouttière interne `$gutter` (4px) rendant les coins de l'indicateur concentriques au track (radius-md 12 = 4 + radius-sm 8), constantes SCSS locales `$sizes`/`$gutter`. API inspirée d'un *select button*
- `ui-toast` : notification transitoire headless en **trois pièces** — la carte présentational **`UiToast`** (icône · titre/texte · fermeture, 100 % tokens `informative`, niveaux `default`/`highlight`/`success`/`warning`/`error` × `subLevel` `high`/`low`, mode bannière `expanded`), le service **`UiToastService`** (`add` retournant l'id / `addAll` / `clear(key?)` / `remove(id)`, store à base de **signals**, `providedIn: 'root'`, inspiré de PrimeNG `MessageService`), et le conteneur **`ui-toast-container`** ancré dans le viewport. Le conteneur lit le service, filtre par `key`, empile les toasts sur **7 positions** (`top`/`bottom` × `left`/`center`/`right` + `center`), anime l'entrée/sortie via le **système de motion** (`animate.enter`/`animate.leave` + `[uiMotion]`, preset choisi selon la position, surchargeable via `motion`/`motionDisabled`), et gère l'**auto-fermeture** (`life`, `sticky` pour la désactiver) avec **mise en pause au survol et au focus clavier**. Messages `UiToastMessage` : `title`/`text`/`level`/`subLevel`/`icon` (nom FA, `true` = icône du niveau, `false` = masquée)/`closable`/`sticky`/`life`/`key`/`data`/`styleClass`. Contenu **personnalisable** via l'input `template` (contexte `{ $implicit: message, closeFn }`) pour les toasts custom et à **actions** (`ui-button`). Positionnement `fixed` par défaut, `contained` (`position: absolute`) pour scoper la pile à un ancêtre positionné. A11y : région live par toast (`role="alert"`+`aria-live="assertive"` pour `error`/`warning`, `role="status"`+`polite` sinon, `aria-atomic`), bouton de fermeture `<button>` natif avec `aria-label`, icône décorative, respect de `prefers-reduced-motion` / `data-motion="off"`. Parité API PrimeNG côté conteneur : `stackVisibleLimit` (plafond de toasts visibles, les autres en file mais avec compte à rebours actif), `stackGap` (écart runtime entre cartes), `preventDuplicates` (fusion des messages de contenu identique), `autoZIndex`/`baseZIndex` (empilement `z-index` automatique/plancher). `breakpoints` volontairement non repris (largeur déjà responsive, styles par palier via `styleClass` + `@media`). API inspirée de PrimeNG `p-toast`. Constantes SCSS locales `$toast-*` (z-index, largeurs, gouttières)
- **Système de motion** partagé, piloté par les tokens et sans dépendance runtime `@angular/animations` : (1) presets d'entrée/sortie globaux `ui-motion-<preset>-enter|leave` (`fade`, `slide-up`/`down`/`left`/`right`, `zoom`, `collapse`) consommés par l'API Angular native `animate.enter`/`animate.leave` (`src/styles/src/base/_motion.scss`, keyframes + classes) ; (2) directive **`[uiMotion]`** (`src/app/shared/motion/ui-motion.ts`) qui sélectionne un preset, expose `enter()`/`leave()` et applique les surcharges de timing (`motionDuration`/`motionEasing`/`motionDelay`/`motionDistance`/`motionScale`/`motionDisabled`) ; (3) mixin SCSS **`utils.motion-transition(...)`** + garde **`utils.motion-reduce`** pour les transitions d'état (`src/styles/src/utils/_motion.scss`). Personnalisation par les custom properties `--ui-motion-*` (défauts = groupe de tokens partagés `$motion-*` dans `_ui-config.scss`). Désactivation par élément (`motionDisabled`), globale (`data-motion="off"` sur `<html>`, réinitialise animations + transitions), et respect automatique de `prefers-reduced-motion`. Documenté dans **Foundations → Motion** (démos live par preset)
- `ui-accordion` : accordéon **headless** en **API de composition** (`ui-accordion` conteneur + `ui-accordion-panel`), inspiré de PrimeNG `p-accordion`. Le conteneur pilote l'état d'ouverture (`value` two-way `model`, modes `single`/`multiple`), la **navigation clavier roving** entre en-têtes (`↑`/`↓` en boucle, `Début`/`Fin`), `selectOnFocus`, et les défauts de groupe (`separator`, `control`, `expandIcon`/`collapseIcon`, `motion`) surchargeables par panneau. En-tête entièrement `<button>` (motif **WAI-ARIA Accordion** : `aria-expanded` + `aria-controls`, cible de clic large), chevron dans une pastille circulaire (Figma 40×40, tokens `actions.low`) réagissant à l'état de l'en-tête ; en-tête riche via le slot `uiAccordionHeader` (ou raccourci `header`). Contenu (`role="region"` + `aria-labelledby`) **conservé dans le DOM** mais `inert` et de hauteur nulle quand fermé (état de formulaire préservé), replié par la technique **`grid-template-rows`** animée via le système de motion (coupable par `[motion]="false"`, reduced-motion et `data-motion="off"`). Panneaux `disabled` (ignorés au clavier), sorties `onOpen`/`onClose`. 100 % tokens `informative`/`actions`, réutilise `ui-separator` et `ui-icon`
- Groupe de tokens de timing partagés `$motion-*` dans `_ui-config.scss` (durées `fast`/`base`/`slow`, easings `standard`/`enter`/`leave`, `distance`, `scale`, catégorie **global**), consommé par le système de motion
- `ui-card` : conteneur **flexible** headless composant des régions optionnelles — **média** pleine largeur (clippé aux coins arrondis), **en-tête** (titre + sous-titre), **contenu** (slot par défaut) et **pied** (actions/tags alignés à droite). Système de slots à double niveau : raccourcis `header` / `subheader` (aucun import) + **directives marqueurs** exportées (`UiCardMedia`, `UiCardTitle`, `UiCardSubtitle`, `UiCardFooter`, forme attribut `[uiCardX]` ou élément `ui-card-x`) pour du contenu riche. Chaque région n'est rendue que si elle porte du contenu (détection via `contentChild` + `afterNextRender` pour le slot par défaut) : aucun wrapper vide ni espacement fantôme. Le slot **titre** centre nativement ses adornments (tag / icône) sur la ligne de texte. Mode **`contentFlush`** : retire la gouttière du contenu (pleine largeur, edge-to-edge) tout en gardant en-tête / pied en retrait — la gouttière reste ré-applicable via la variable CSS exposée `--ui-card-padding` (ex. bande média/couleur sans gutter + paragraphe gouttiéré). Le pied s'aligne à droite par défaut mais accueille la grille **Gridaflex** (`flex-x` / `flex-gap-x` / `align-justify`) pour l'espacement et la répartition (tags à gauche / action à droite). Variantes de surface `outlined` / `elevated` / `flat`, promotion en `role="region"` via `ariaLabel` / `ariaLabelledBy`. API inspirée de PrimeNG `p-card`, 100 % tokens `global` (light/dark), personnalisation via variables SCSS locales `$card-*`
- `ui-textarea` : zone de texte **multiligne** headless — bâtie sur le shell `ui-field` (mode `multiline`) + un `<textarea>` natif. Label + helper, niveaux `default`/`success`/`error` (bordure + message), tailles `default`/`small`, `rows`, `maxlength`, `placeholder`, `required`/`disabled`/`readonly`/`invalid`. Redimensionnement via la poignée **native** logée dans le coin bas-droit — axe configurable `resize` (`none`/`vertical`/`horizontal`/`both`, `vertical` par défaut) — ou `autoResize` (la hauteur suit le contenu). Compteur de caractères optionnel (`showCount`, affiche `saisi / max` avec `maxlength`). Gestion d'erreur **dynamique** via la validation du contrôle (`showError` = invalide + touché/modifié). Valeur via `ControlValueAccessor` (`BaseFormField`) compatible reactive forms et `[(ngModel)]`. API inspirée de PrimeNG `pTextarea` (`autoResize`, `rows`). Co-localisé, 100 % tokens `form`.
- Mode `multiline` du shell `ui-field` : boîte haut-alignée sans padding (le contenu porte son inset, poignée de resize dans le coin), hauteur automatique avec `min-height`, et **ligne de pied de champ** (`hasFooter` + slot `[uiFieldFooter]`) pour héberger un compteur (support de `ui-textarea`)
- Groupe de constantes partagées `$form-textarea-*` dans `_ui-config.scss` (min-height + padding vertical, catégorie **forms**)
- `ui-rating` : notation étoiles headless — configuration du nombre d'étoiles, annulation (toggle off au clic), support natif du clavier et des lecteurs d'écran via un `<input type="range">` invisible superposé (pattern WAI-ARIA slider), hover interactif sur les étoiles SVG (`ui-icon`). Valeur via `ControlValueAccessor` compatible avec les formulaires réactifs et template-driven, gestion des états `disabled` / `readonly` / `invalid`. Co-localisé, 100 % design tokens `form`.
- `ui-datepicker` : sélecteur de **date / mois / année** (et d'**heure** optionnelle) headless — champ déclencheur `ui-input` en lecture seule ouvrant un calendrier dans un overlay Angular CDK (ou rendu **inline**). Sélection `single` / `multiple` / `range` (`selectionMode`, valeur `Date | Date[] | null`), **vues à tiroir** jour → mois → année (clic sur l'entête) + modes `MonthPicker`/`YearPicker` (`view`), **plusieurs mois** côte à côte (`numberOfMonths`, séparateur + titres alignés sur les flèches), ouverture en overlay avec **retournement automatique** au-dessus quand la place manque (désactivable via `autoFlip`), navigation mensuelle, **focus roving** clavier complet (flèches / Début-Fin / Page↑↓ ±année / Entrée-Espace / Échap, motif WAI-ARIA), bornes `minDate`/`maxDate`, `disabledDays`/`disabledDates`, `showTime` (steppers verticaux HH:MM façon PrimeNG, pattern ARIA `spinbutton` pilotable au clavier, `hourFormat` 12/24h + AM/PM), `timeOnly` (icône horloge), effacement `showClear` (croix dans le champ), barre de boutons `showButtonBar` (Aujourd'hui/Effacer), `firstDayOfWeek`, `locale` + `dateFormat` custom, `closeOnSelect`. **Templates** projetés `#date` (cellule de jour, contexte `$implicit`+`selected`) et `#buttonbar` (contexte `todayCallback`/`clearCallback`). Valeur via `ControlValueAccessor` sur `BaseFormField` (label/helper/erreur/niveaux/états partagés). 100 % tokens `form`/`actions`, overlay ancré sur la boîte de champ (placement correct même avec helper), personnalisation via `panelStyleClass` + variables SCSS locales. API inspirée de PrimeNG `p-datepicker`
- `ui-tag` : étiquette informative (pilule) headless — libellé + icône optionnelle de chaque côté (`iconLeft`/`iconRight`), familles de couleur `level` (`default`/`highlight`/`success`/`warning`/`error`) × intensité `subLevel` (`high`/`low`), tailles `default`/`small`, forme `rounded` (pilule) ou rectangle arrondi. Composant statique inspiré de l'API PrimeNG `Tag` (`severity` → `level`×`subLevel`), 100 % tokens `informative`, garde-fou a11y icon-only (`ariaLabel`)

### Changed
- `ui-datepicker` : la coque du panneau consomme désormais les constantes partagées `$overlay-panel-*` (`$panel-radius`/`$panel-padding`/`$panel-shadow` retargetées — valeurs identiques, zéro changement visuel)
- `ui-button` : **hôte polymorphe** — quand `href` ou `routerLink` est fourni, le composant rend un **vrai `<a>`** stylé comme un bouton (au lieu du `<button>`), pour couvrir les CTA qui **naviguent** tout en préservant le comportement natif de l'ancre (Cmd/Ctrl+clic, clic-molette, « ouvrir / copier le lien ») — aucune navigation JS bricolée dans un `(click)`. Nouveaux inputs `href`/`routerLink`/`target`/`rel` (rel `noopener noreferrer` par défaut si `target="_blank"`) ; `disabled` en mode lien retire `href`/`routerLink`, pose `aria-disabled` et sort de la tabulation (pas de `disabled` natif sur `<a>`). Contenu, icônes, tailles et niveaux inchangés. `role="button"` conservé sur l'ancre. Le choix de l'élément suit la **sémantique** (naviguer ⇒ `<a>`, agir ⇒ `<button>`), pas l'apparence ; pour un lien **inline**, utiliser `ui-link`
- Doc de configuration (`Components/Configuration`) refondue : la page principale devient « fonctionnement + sommaire », les groupes de variables sont éclatés en sous-pages (`Global UI` / `Forms` / `Actions` / `Informative`) et chaque composant reçoit une section « Configuration (SCSS) » listant ses variables locales
- `ui-toggle` : ajout d'un template de curseur personnalisé `handle` (contexte `{ checked }`, façon PrimeNG) et de `labelPosition` (`before`/`after`, label cliquable des deux côtés)
- `BaseControlValueAccessor` complété (setDisabledState, état `showError`, désabonnement automatique)
- `ui-icon` : boîte **carrée adaptative** (`aspect-ratio: 1`) pour un encombrement prévisible quelle que soit l'avance du glyphe FontAwesome
- Explorateur `Foundations/Colors` : bascule marque + mode en direct, y compris en mode clair
