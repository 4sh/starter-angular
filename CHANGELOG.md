# Changelog

Toutes les évolutions notables de ce Design System sont documentées dans ce fichier.

Le format suit [Keep a Changelog](https://keepachangelog.com/fr/1.1.0/) et le projet adhère à [Semantic Versioning](https://semver.org/) (voir [VERSIONING.md](./VERSIONING.md)).

> Phase de construction des fondations : tout reste versionné `0.1.0`. Le versioning
> sémantique sera activé une fois la base stabilisée.

## [Unreleased]

### Fixed
- `ui-button` : suppression d'un `NG0951` (« Child query result is required but no value is available ») émis à chaque rendu. L'effet de forwarding des `buttonProps` lisait le `viewChild.required('#host')` dans un `effect()` nu, exécuté pendant `runEffectsInView` avant la résolution de la requête de vue (le `#host` est sous `@if/@else`). Basculé en `afterRenderEffect` (browser-only, vue garantie résolue) avec court-circuit quand `buttonProps` est absent : plus d'erreur dans le `ErrorHandler`, comportement de forwarding inchangé

### Fixed
- `ui-table` : **virtual scroll — boucle de défilement infinie** — le *scroll anchoring* du navigateur combattait le fenêtrage : chaque changement de hauteur de la cale supérieure « ré-ancrait » la position de scroll, ce qui déclenchait un nouvel événement scroll, déplaçait la fenêtre, et ainsi de suite sans fin ; `overflow-anchor: none` sur le viewport en mode `virtualScroll` coupe la boucle (comportement de scroll normal et borné)
- `ui-table` : **colonnes figées × en-tête sticky** — en mode `scrollable`, les `th` non figés (z-index 2, règle plus spécifique) repassaient au-dessus des `th` figés lors du scroll horizontal ; la règle `th._frozen-*` du bloc scrollable rétablit la hiérarchie (cellules figées 1 < en-tête 2 < en-tête figé 3)
- `ui-table` : **interstice des lignes figées** — 1px de contenu défilant transparaissait entre deux lignes sticky : les offsets cumulés arrondissaient au plus proche alors que les hauteurs de lignes sont fractionnaires ; les tops sont désormais **arrondis à l'entier inférieur** (léger chevauchement < 1px, recouvert par la ligne suivante, au lieu d'un jour)

### Changed
- `ui-table` : **état de chargement repensé** — plus de masque translucide au-dessus de l'en-tête : pendant `loading`, les lignes du corps sont remplacées par une **ligne d'attente** (`ui-spinner` centré, colspan complet), l'en-tête et le pied restant nets et en place ; les lignes apparaissent une fois les données chargées (a11y inchangée : `aria-busy` racine + spinner `role="status"` nommé ; la custom property `--ui-table-mask-background` disparaît au profit de la constante locale `$loading-padding-y`)
- `ui-table` : **anneau de focus des lignes** — l'indicateur `:focus-visible` encadre désormais la `<tr>` entière (outline intérieur) au lieu d'un box-shadow inset par cellule (qui dessinait quatre bordures par `<td>`) ; **pagination déléguée** au nouveau composant autonome `ui-paginator` (rendu et API `pageChange` inchangés)
- `ui-field` : **mode `autoHeight`** — la boîte grandit avec un contenu qui passe à la ligne (min-height = hauteur fixe, padding vertical `xs`, centrage conservé ; rendu identique sur une seule ligne). Consommé par `ui-select` en mode `multiple` pour le wrap des chips/valeurs
- `ui-icon` : **hook `--ui-icon-size`** — la taille rendue est désormais lue via `var(--ui-icon-size, <px du preset>)` sur chaque classe de taille. Un composant hôte peut ainsi rendre une icône **au-delà du preset `xl` (32px)** — ex. une icône hero dans `ui-empty-state` — en posant `--ui-icon-size` sur le `<ui-icon>` ou un ancêtre : les custom properties héritent **au travers de la frontière d'encapsulation**, ce qui évite tout `::ng-deep`. Comportement par défaut inchangé (repli sur la valeur du preset `size` quand le hook n'est pas défini)
- `ui-button` : **hook `--ui-button-radius`** — le rayon des coins est désormais lu via `var(--ui-button-radius, var(--radius-sm))`, permettant à un composant hôte de reformer les coins sans percer l'encapsulation (`::ng-deep`). Consommé par `ui-button-split` pour souder ses deux boutons ; comportement par défaut inchangé (repli sur `--radius-sm`)

- `ui-slider` : **curseur de sélection de valeur** headless (catégorie *forms*, `ControlValueAccessor`) — choix d'une valeur numérique (ou d'un intervalle) en glissant une poignée le long d'une piste. Bâti sur le **motif WAI-ARIA « slider »** : chaque poignée est un élément `role="slider"` exposant `aria-valuemin`/`aria-valuemax`/`aria-valuenow`/`aria-orientation`, entièrement pilotable au **clavier** (flèches ± `step`, Page↑/↓ ± 10 pas, Début/Fin → `min`/`max`). Outputs **`sliderChange`** (émis en continu pendant le déplacement) et **`slideEnd`** (émis une fois le glissement/clic terminé), plus `sliderFocus`/`sliderBlur`. Style erreur automatique quand le contrôle attaché est invalide et touched/dirty (`invalid` pour forcer) ; états `hover`/`:focus-visible`/`pressed`/`disabled` en pseudo-classes sur les tokens `actions.*`/`form.error.*` (piste `global.low.surface`, remplissage/poignées `actions.high.surface`, repères `informative.defaulthigh`/`actions.high.content`). **Hooks structurels** CSS custom properties surchargeables par instance : `--ui-slider-thickness`, `--ui-slider-handle-size`, `--ui-slider-length` (vertical). Avertissement dev si aucun nom accessible. Compatible autonome, `[(ngModel)]`, Reactive Forms et **Signal Forms** (`[field]`, prêt pour la migration Angular 22). **Divergences assumées** vs YAML Figma : la prop `Type` (Default/Marks/Range) est décomposée en inputs fonctionnels `range` + `marks` ; la prop `Level` (1-4, largeurs de piste fixes) devient une **largeur fluide** (fill container) conforme aux règles Auto Layout ; taille des repères portée de 3px à `rem-calc(6)` pour la lisibilité. ### Changed
- `ui-link` : nouvel input `ariaCurrent` — pose `aria-current` sur l'ancre (ex. `"page"` pour l'élément courant d'un fil d'Ariane ; consommé par `ui-breadcrumb`)
- `ui-datepicker` : la coque du panneau consomme désormais les constantes partagées `$overlay-panel-*` (`$panel-radius`/`$panel-padding`/`$panel-shadow` retargetées — valeurs identiques, zéro changement visuel)
- `ui-button` : **hôte polymorphe** — quand `href` ou `routerLink` est fourni, le composant rend un **vrai `<a>`** stylé comme un bouton (au lieu du `<button>`), pour couvrir les CTA qui **naviguent** tout en préservant le comportement natif de l'ancre (Cmd/Ctrl+clic, clic-molette, « ouvrir / copier le lien ») — aucune navigation JS bricolée dans un `(click)`. Nouveaux inputs `href`/`routerLink`/`target`/`rel` (rel `noopener noreferrer` par défaut si `target="_blank"`) ; `disabled` en mode lien retire `href`/`routerLink`, pose `aria-disabled` et sort de la tabulation (pas de `disabled` natif sur `<a>`). Contenu, icônes, tailles et niveaux inchangés. `role="button"` conservé sur l'ancre. Le choix de l'élément suit la **sémantique** (naviguer ⇒ `<a>`, agir ⇒ `<button>`), pas l'apparence ; pour un lien **inline**, utiliser `ui-link`
- Doc de configuration (`Components/Configuration`) refondue : la page principale devient « fonctionnement + sommaire », les groupes de variables sont éclatés en sous-pages (`Global UI` / `Forms` / `Actions` / `Informative`) et chaque composant reçoit une section « Configuration (SCSS) » listant ses variables locales

- `BaseControlValueAccessor` complété (setDisabledState, état `showError`, désabonnement automatique)
