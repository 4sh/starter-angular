## Principe : 3 niveaux, une variable locale par composant

`_ui-config.scss` (exposé par <code>@use 'utils'</code>) déclare des constantes à 3 portées,
de la plus large à la plus étroite :

```
GLOBAL UI  ──►  CATÉGORIE  ──►  COMPOSANT (variable locale)
(tous)          (forms /        (un seul .scss)
                 actions)
```

- Chaque variable de **catégorie** hérite par défaut de la valeur **globale**.
- Chaque **composant** lit la valeur de catégorie dans une **variable locale**, en tête de
  son `.scss`. C'est ce qui préserve l'indépendance : ajuster un composant = changer sa
  variable locale, sans impacter les autres.

```scss
// Dans ui-radio.scss : la valeur commune arrive via une variable LOCALE :
$focus-ring-width: utils.$form-focus-ring-width; // ← remplacer ici = ajuster CE composant
```

## Comment surcharger

<table style={{ width: '100%', borderCollapse: 'collapse', fontSize: 14 }}>
  <thead>
    <tr style={{ textAlign: 'left', borderBottom: '2px solid rgba(128,128,128,.25)' }}>
      <th style={{ padding: 8 }}>Objectif</th>
      <th style={{ padding: 8 }}>Où intervenir</th>
    </tr>
  </thead>
  <tbody>
    <tr style={{ borderBottom: '1px solid rgba(128,128,128,.15)' }}>
      <td style={{ padding: 8 }}><strong>Tout le kit</strong> (ex : focus ring de 2px → 3px partout)</td>
      <td style={{ padding: 8 }}>Éditer la valeur globale dans <code>_ui-config.scss</code>. Tous les composants re-<code>@use</code> ce fichier, ils suivent.</td>
    </tr>
    <tr style={{ borderBottom: '1px solid rgba(128,128,128,.15)' }}>
      <td style={{ padding: 8 }}><strong>Une catégorie</strong> (ex : contrôles de formulaire plus grands)</td>
      <td style={{ padding: 8 }}>Éditer la variable `$form-*` (ou `$action-*`) dans <code>_ui-config.scss</code>.</td>
    </tr>
    <tr style={{ borderBottom: '1px solid rgba(128,128,128,.15)' }}>
      <td style={{ padding: 8 }}><strong>Un seul composant</strong> (ex : anneau plus épais sur le radio uniquement)</td>
      <td style={{ padding: 8 }}>Remplacer la valeur de la variable locale en tête du <code>.scss</code> du composant.</td>
    </tr>
  </tbody>
</table>

> Deux leviers, c'est tout. Les composants font <code>@use 'utils'</code> (qui expose
> `ui-config`) et lisent chaque valeur dans une variable **locale** ; éditer `_ui-config.scss`
> se répercute donc partout, et remplacer la variable locale n'affecte que le composant.

## Mixin `control-transition()`

Applique la recette de transition mutualisée à une liste de propriétés (durée + easing pris
dans `ui-config`), en une déclaration :

```scss
@use 'utils';

.ui-checkbox-box {
  @include utils.control-transition(background-color, border-color, box-shadow);
}
// → transition: background-color 80ms ease-in-out, border-color 80ms ease-in-out, box-shadow 80ms ease-in-out;
```

Changer la durée/courbe pour tous les composants = éditer `$control-transition-duration` /
`$control-transition-easing` dans `ui-config`.

## Gouvernance : quand créer une constante partagée

- Valeur utilisée par **≥ 2 composants** → la remonter dans `ui-config` (niveau **catégorie**
  si propre à `forms`/`actions`, **global** sinon).
- Valeur d'un **seul** composant → elle reste **locale** à son `.scss`.
- `ui-config` porte des **choix de tokens** (structure). Les couleurs thémables restent des
  **design tokens** runtime : **jamais** de couleur ici.
