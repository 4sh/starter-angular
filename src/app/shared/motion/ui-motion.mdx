# Motion

Système d'animation **léger, piloté par les design tokens** et pensé pour être
**activable, personnalisable et réutilisable** sur tous les composants (ouverture
d'accordéon, dropdown, modale, apparition d'un helper ou d'un toast…).

Il s'appuie sur les **animations CSS natives** et sur l'API Angular native
`animate.enter` / `animate.leave` (aucune dépendance runtime `@angular/animations`).
Deux primitives complémentaires :

- **Presets d'entrée/sortie** — des classes globales `ui-motion-<preset>-enter|leave`
  pour l'insertion/retrait d'un élément (`@if`, `@for`, overlays, toasts…).
- **Transitions d'état** — le mixin SCSS `utils.motion-transition(...)` pour un élément
  monté qui bascule (accordéon replié/déplié, tooltip visible/masqué…).

## Utilisation

### 1. Directive `[uiMotion]` (recommandé)

Sur un élément rendu conditionnellement, la directive choisit le preset, expose les
classes `enter()`/`leave()` et applique les surcharges de timing :

```html
@if (open()) {
  <div class="panel"
       [uiMotion]="'zoom'" #m="uiMotion"
       [animate.enter]="m.enter()" [animate.leave]="m.leave()">…</div>
}
```

### 2. Classes globales (CSS-only)

Les mêmes classes sont utilisables directement avec `animate.enter`/`animate.leave` :

```html
@if (open()) {
  <div animate.enter="ui-motion-slide-down-enter"
       animate.leave="ui-motion-slide-down-leave">…</div>
}
```

### 3. Transition d'état (élément monté)

```scss
@use 'utils';
.panel {
  opacity: 0;
  @include utils.motion-transition(opacity, transform);
  &._open { opacity: 1; }
  @include utils.motion-reduce { transition-duration: 0.01ms; }
}
```

## Personnalisation

Chaque élément se re-cadence via les **custom properties** `--ui-motion-*` (par la
directive : `motionDuration`, `motionEasing`, `motionDelay`, `motionDistance`,
`motionScale`, ou en CSS directement), avec les **tokens** `$motion-*` comme défauts :

<table className="doc-table">
  <thead>
    <tr><th>Custom property</th><th>Défaut (token)</th><th>Rôle</th></tr>
  </thead>
  <tbody>
    <tr><td>`--ui-motion-duration`</td><td>`--transition-default` (180ms)</td><td>Durée.</td></tr>
    <tr><td>`--ui-motion-easing`</td><td>`--transition-easeinout`</td><td>Courbe (transitions symétriques).</td></tr>

## Désactivation & accessibilité

- **Par élément** : `[motionDisabled]="true"` (directive) ou `[motion]="false"` (accordéon).
- **Global** : `data-motion="off"` sur `<html>` neutralise toutes les animations **et**
  transitions du kit (même réinitialisation que reduced-motion).
- **Préférence système** : `prefers-reduced-motion: reduce` réduit tous les presets à un
  flash instantané — l'état change toujours, sans mouvement.
