# ui-tabs

Onglets **headless** : bascule entre plusieurs vues. API de **composition** , un
conteneur `ui-tabs` qui pilote l'état, une liste `ui-tab-list` d'onglets `ui-tab`,
et un `ui-tab-panels` regroupant les `ui-tab-panel`. Chaque onglet et son panneau
sont appariés par leur `value`.

100 % piloté par les **design tokens** (`navigation.*` pour la liste, `global` pour
le contenu). L'indicateur d'onglet actif glisse , et chaque panneau apparaît en
fondu , via le **système de motion** partagé, qui respecte automatiquement
`prefers-reduced-motion` et le commutateur global `data-motion="off"`.

## Défilement

`scrollable` transforme la liste en zone défilante ; les boutons **précédent /
suivant** (`showNavigators`, par défaut visibles) apparaissent quand la liste
déborde.

<Canvas of={TabsStories.Scrollable} />

## Activation au focus

Avec `selectOnFocus`, déplacer le focus au clavier (flèches) **active** l'onglet
au passage (activation automatique du motif WAI-ARIA Tabs). Sinon, l'activation
reste manuelle (`Entrée` / `Espace`).

<Canvas of={TabsStories.SelectOnFocus} />

## Chargement paresseux

Par défaut le contenu d'un onglet inactif est **rendu** (mais masqué). `lazy` (sur
`ui-tabs` pour tout le groupe, ou sur un `ui-tab-panel`) ne rend le contenu **qu'à
la première activation**. Pour n'initialiser des composants complexes qu'à ce
moment, enveloppez le contenu dans `<ng-template #content>…</ng-template>`.

## Accessibilité

<table className="doc-table">
  <thead>
    <tr><th>Point</th><th>Implémentation</th></tr>
  </thead>
  <tbody>
    <tr>
      <td>Boutons natifs</td>
      <td>Chaque onglet est un `&lt;button role="tab"&gt;` natif (l'hôte `ui-tab` est `display: contents`), donc focusable et actionnable au clavier ; `disabled` natif.</td>
    </tr>
    <tr>
      <td>Structure ARIA</td>
      <td>`role="tablist"` (avec `aria-orientation`) → `role="tab"` (`aria-selected`, `aria-controls`) → `role="tabpanel"` (`aria-labelledby` vers son onglet, `tabindex="0"`).</td>
    </tr>
    <tr>
      <td>Clavier</td>
      <td>`←`/`→` (horizontal) ou `↑`/`↓` (vertical) déplacent le focus entre onglets (roving, boucle) ; `Début`/`Fin` au premier/dernier ; `Entrée`/`Espace` activent. `selectOnFocus` active au déplacement.</td>
    </tr>
    <tr>
      <td>Tab stop unique</td>
      <td>Seul l'onglet actif (ou le premier disponible) est dans l'ordre de tabulation (roving `tabindex`) ; les autres sont atteints par les flèches.</td>
    </tr>
    <tr>
      <td>Panneau masqué</td>
      <td>Le panneau inactif reste dans le DOM mais est `hidden` (état conservé, invisible pour les lecteurs d'écran).</td>
    </tr>
    <tr>
      <td>Icône décorative</td>
      <td>`ui-icon` en `aria-hidden` ; le sens passe par le libellé, pas seulement par l'icône.</td>
    </tr>
    <tr>
      <td>Motion</td>
      <td>Indicateur et fondu neutralisés sous `prefers-reduced-motion`, `data-motion="off"` et `[motion]="false"`.</td>
    </tr>
  </tbody>
</table>

## Exemples

```html
<!-- Base -->
<ui-tabs [(value)]="active">
  <ui-tab-list ariaLabel="Sections">
    <ui-tab [value]="0">Aperçu</ui-tab>
    <ui-tab [value]="1">Activité</ui-tab>
  </ui-tab-list>
  <ui-tab-panels>
    <ui-tab-panel [value]="0">Contenu 1…</ui-tab-panel>
    <ui-tab-panel [value]="1">Contenu 2…</ui-tab-panel>
  </ui-tab-panels>
</ui-tabs>

<!-- Vertical + scrollable + chargement paresseux -->
<ui-tabs [(value)]="active" orientation="vertical" [scrollable]="true" [lazy]="true"> … </ui-tabs>

<!-- Menu de navigation (onglets seuls + router-outlet) -->
<ui-tabs [value]="url">
  <ui-tab-list ariaLabel="Navigation">
    <ui-tab [value]="'/dashboard'" icon="gauge">Tableau de bord</ui-tab>
    <ui-tab [value]="'/team'" icon="users">Équipe</ui-tab>
  </ui-tab-list>
</ui-tabs>
<router-outlet />
```
