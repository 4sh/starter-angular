# ui-sidebar

Barre latérale applicative :

- **Statique** (défaut) : la barre vit dans le flux et **pousse le contenu**. Repliable en
  **rail d'icônes** (`collapsed`, two-way). Avec `openOnHover`, un rail replié **flotte
  temporairement** ouvert au survol / focus, sans décaler le contenu.
- **Overlay** : la barre **flotte** au-dessus du contenu en offcanvas avec un `backdrop`
  optionnel, piloté par le modèle two-way `visible` — **piège de focus** + restauration
  (`@angular/cdk/a11y`), `aria-modal`, **blocage du scroll**, `Échap` et clic extérieur.

## Avec menu

Chrome applicatif complet : sélecteur d'espace de travail en en-tête (`#header`), navigation
groupée via `ui-sidebar-menu`, carte utilisateur en pied (`#footer`). Le déclencheur replie la
barre en rail d'icônes — libellés, en-têtes de sections et badges masqués.

Les templates `#header` / `#footer` reçoivent l'**état replié résolu** dans leur contexte
(`let-collapsed="collapsed"`), pour n'afficher que les icônes en mode rail :

```html
<ng-template #header let-collapsed="collapsed">
  <ui-icon name="layer-group" size="lg" />
  @if (!collapsed) { <strong>App</strong> }
</ng-template>
```

## Responsive

Sous `breakpoint` (ici `1024px`, aligné sur l'échelle `$breakpoint-*` du design system), la
barre devient un offcanvas avec fond ; au-dessus, elle reste un rail d'icônes qui pousse le
contenu. Redimensionnez l'aperçu pour observer la bascule.

## Double barre

Chrome à deux niveaux façon console d'administration : un **rail d'icônes** permanent (replié,
libellés en info-bulle) comme navigation primaire, accolé à une **barre de navigation
secondaire** listant les sections d'un module, le tout bordant le contenu. Deux `ui-sidebar`
statiques côte à côte — pas d'offcanvas.

## Menu imbriqué

Les entrées deviennent des groupes **repliables** révélant un sous-arbre imbriqué. L'**état
actif est suivi** : un groupe contenant l'élément actif se déplie automatiquement.

## ui-sidebar-menu

Menu de navigation déclaratif (`UiSidebarMenuItem[]`) rendu comme **landmark de navigation** :
un `<nav>` enveloppant des conteneurs `role="list"` / `role="listitem"`. Il gère :

- des **sections** libellées (navigation groupée) ;
- des **groupes repliables** (`toggleable`) révélant des sous-arbres profonds, avec état
  d'expansion (`expandedKeys` two-way) ;
- le **suivi d'état actif** par item (RouterLink pour les liens de route, `active` pour la
  navigation manuelle — les groupes contenant un descendant actif se déplient d'office) ;
- un **badge** de fin rendu via `ui-badge` (`badge` + `badgeLevel`) et une icône « ouvre
  ailleurs » automatique pour les liens externes (`external` ou `target="_blank"`).

Déposé dans un `ui-sidebar`, il lit l'état replié résolu par DI et se replie en **rail
d'icônes** (libellés masqués, boutons parfaitement carrés). En usage autonome, pilotez le rail
avec l'entrée `collapsed`. En rail, activez **`tooltips`** pour révéler le libellé de chaque
icône au survol / focus (via la directive `[uiTooltip]`, positionnée à droite).

## uiSidebarTrigger

Directive qui **bascule** une barre depuis n'importe quel élément. Elle reflète le contrat
d'accessibilité entre un contrôle et la barre : `aria-controls` (id du panneau) et
`aria-expanded` (état résolu ouvert / replié), et bascule à l'activation. `Entrée` / `Espace`
sont natifs sur un `<button>`.

```html
<button [uiSidebarTrigger]="sb" aria-label="Basculer le menu">☰</button>
<ui-sidebar #sb>…</ui-sidebar>
```
