# ui-menu

Menu de **navigation / commandes** headless, statique ou **popup**, piloté par un
modèle déclaratif `items` (`UiMenuItem[]`) : groupes à en-tête, séparateurs,
**sous-menus repliables**, commandes (`command`), liens routeur (`routerLink`)
et URL externes (`url`).

<Canvas of={MenuStories.Basic} />

## API

<ArgTypes of={MenuStories} />

Méthodes publiques (mode `popup`) : `toggle(event)`, `show(event)`, `hide(focusTrigger?)`.
Sorties : `opened` / `closed` (popup) et `itemClick` (`{ originalEvent, item }`).

## Groupes & sélection pilotée par le modèle

Les items sont groupés par la propriété `items` (en-tête + séparateur plat entre
sections). Ici, cases à cocher et radios sont **dérivées du modèle** : le
`command` bascule l'état, et l'icône de l'item est recalculée depuis ce même état.

<Canvas of={MenuStories.Group} />

> Deux rendus de groupes via l'input `submenus` : `inline` (sections / accordéons
> ci-dessous, le défaut) ou `flyout` (panneaux latéraux en cascade — le mode
> utilisé par [`ui-context-menu`](?path=/docs/components-ui-navigation-ui-context-menu--docs) :
> survol/clic ou `→`/`←` au clavier, parents `aria-haspopup` + `aria-expanded`).

## Sous-menus repliables (`toggleable`)

Les groupes **imbriqués** sont repliables par défaut. `toggleable` force le
comportement par item : ici le groupe **racine** « Fichiers » est rendu repliable,
et le groupe imbriqué « Partagés » est **figé ouvert** (`toggleable: false`).
Le repli est animé par le système de motion (technique grid-rows, comme
l'accordéon), coupé par `[motion]="false"` / `prefers-reduced-motion`.

<Canvas of={MenuStories.Toggleable} />

## Popup

`popup` + la méthode `toggle(event)` : le panneau s'ouvre dans un **overlay CDK**
ancré sous l'élément déclencheur (retournement automatique au-dessus quand la
place manque — `autoFlip`), avec le preset de motion `slide-down`. Fermeture au
clic extérieur, à `Échap` (focus rendu au déclencheur) et après activation d'un item.

```html
<ui-button label="Options" (buttonClick)="menu.toggle($event)"
  [buttonProps]="{ 'aria-haspopup': 'menu', 'aria-controls': menu.uid }" />
<ui-menu #menu popup [items]="items" ariaLabel="Options" />
```

## Compact (`size="small"`)

Densité réduite pour les **menus d'actions contextuels** — typiquement un bouton
icon-only « … » sur une ligne de liste, une carte ou une cellule de tableau.

<Canvas of={MenuStories.Compact} />

## Templates

`#item` personnalise le **contenu** de chaque menuitem (contexte `$implicit` =
l'item du modèle), `#submenuheader` celui des en-têtes de groupe ; les slots
`#start` / `#end` encadrent la liste.

## Controlled (`expandedKeys`)

`expandedKeys` définit les groupes repliables ouverts : un objet dont la **clé
est l'`id` de l'item** et la valeur un booléen. La propriété est **two-way
bindable** (`[(expandedKeys)]`). Sans `id`, une clé positionnelle interne est
générée — stable tant que le tableau `items` ne change pas, mais donnez un `id`
dès que vous contrôlez l'état de l'extérieur.

## Accessibilité

<table className="doc-table">
  <thead>
    <tr><th>Point</th><th>Implémentation</th></tr>
  </thead>
  <tbody>
    <tr><td>Rôles</td><td>`role="menu"` → `role="menuitem"` sur de **vrais** `<a>` / `<button>` (jamais de `div` cliquable) ; sections `role="group"` + `aria-labelledby`, séparateurs `role="separator"`.</td></tr>
    <tr><td>Clavier</td><td>Motif menu WAI-ARIA : **tab stop unique** (roving tabindex), `↑`/`↓` en boucle, `Début`/`Fin`, `Entrée`/`Espace` activent, `→`/`←` ouvrent/ferment un groupe repliable, `Échap` ferme le popup.</td></tr>
    <tr><td>Sous-menus</td><td>Toggle `aria-expanded` + `aria-controls` ; contenu replié `inert` (invisible au clavier et aux lecteurs d'écran).</td></tr>
    <tr><td>Focus</td><td>`:focus-visible` distinct du hover (anneau `--global-high-stroke-focused`, la famille `navigation` n'ayant pas d'état `focused`). Ouverture popup → focus sur le premier item ; fermeture → retour au déclencheur.</td></tr>
    <tr><td>Désactivé</td><td>`disabled` natif sur `<button>` (une entrée lien désactivée retombe sur un `<button>` désactivé), tokens `-disabled`, sautée par la navigation clavier.</td></tr>
    <tr><td>Nom accessible</td><td>`ariaLabel` / `ariaLabelledBy` sur la liste ; `ariaLabel` par item si besoin.</td></tr>
  </tbody>
</table>
