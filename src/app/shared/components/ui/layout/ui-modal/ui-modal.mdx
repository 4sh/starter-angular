# ui-modal

Fenêtre modale **headless** : une surface flottante stylée par tokens, rendue au-dessus
d'un masque de viewport et pilotée par le modèle two-way `visible`. L'apparition / la
disparition passent par le **système de motion** (fondu du masque + `zoom` du dialogue par
défaut, surchargeable via `motion`). Pensée pour l'accessibilité : **piège de focus** avec
restauration (`@angular/cdk/a11y`), `aria-modal`, **blocage du scroll de fond**, fermeture
au clavier (`Échap`), et fermeture optionnelle au clic extérieur.

Le dialogue compose trois régions : un `header` (chaîne simple ou template `#header`), le
contenu par défaut (`<ng-content>`, qui **scrolle en interne** si long) et un pied via le
template `#footer`. Il gère `draggable`, `maximizable`, neuf positions, le mode modal /
non-modal et des largeurs par palier (`breakpoints`).

## Contenu & templates

- `header` (chaîne) ou `#header` (contenu riche) pour le titre.
- `<ng-content>` pour le corps.
- `#footer` pour les actions.
- `#closeicon` pour remplacer l'icône de fermeture.

## Déplacement

`draggable` active la saisie du dialogue par son en-tête (les boutons d'action ne
déclenchent pas de déplacement). `keepInViewport` (défaut) le maintient dans les limites de
l'écran ; `minX` / `minY` bornent le coin haut-gauche. Le déplacement s'applique via la
propriété CSS indépendante `translate`, sans interférer avec l'animation `zoom`.

## Redimensionnement

`resizable` ajoute une poignée au coin bas-droit du dialogue (pointeur uniquement, à la
manière d'un coin de `textarea`). Les tailles minimales sont bornées par `minWidth` /
`minHeight`. Pour un contrôle au clavier de la taille, utiliser `maximizable`.

<Canvas of={ModalStories.Resizable} />

## Embarqué (`contained`)

`contained` scope le dialogue à l'ancêtre positionné le plus proche (`position: absolute`
au lieu de `fixed`) et **n'affecte pas** le scroll du body : idéal pour intégrer une modale
dans un conteneur borné (panneau, aperçu). Le conteneur parent doit être positionné
(`position: relative`).

<Canvas of={ModalStories.Contained} />

## Modal / non-modal

En mode `modal` (défaut), le masque assombrit et **bloque le scroll** de l'arrière-plan ;
`dismissableMask` autorise la fermeture au clic extérieur. En mode non-modal, l'arrière-plan
reste **interactif et scrollable** (`blockScroll` peut tout de même figer le scroll).

## Accessibilité

<table className="doc-table">
  <thead>
    <tr>
      <th>Règle</th>
      <th>Détail</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Rôle & nom</strong></td>
      <td><code>role="dialog"</code> + <code>aria-modal="true"</code>. Nom accessible via <code>aria-labelledby</code> (titre de l'en-tête) sinon <code>ariaLabel</code> / <code>ariaLabelledBy</code>. Un warning dev signale un dialogue sans nom accessible.</td>
    </tr>
    <tr>
      <td><strong>Focus</strong></td>
      <td>Piège de focus (<code>cdkTrapFocus</code>) tant que le dialogue est ouvert ; le focus est capturé à l'ouverture (<code>focusOnShow</code>) et <strong>restauré</strong> sur l'élément déclencheur à la fermeture.</td>
    </tr>
    <tr>
      <td><strong>Clavier</strong></td>
      <td><code>Échap</code> ferme (<code>closeOnEscape</code> + <code>closable</code>). Boutons d'en-tête <code>&lt;button&gt;</code> natifs avec <code>aria-label</code> ; l'agrandissement expose <code>aria-pressed</code>.</td>
    </tr>
    <tr>
      <td><strong>Scroll de fond</strong></td>
      <td>Bloqué quand <code>modal</code> (ou <code>blockScroll</code>), avec compensation de la gouttière de scrollbar (pas de saut de mise en page). Compteur partagé : le scroll est rétabli à la fermeture du dernier dialogue.</td>
    </tr>
    <tr>
      <td><strong>Clic extérieur</strong></td>
      <td>Optionnel via <code>dismissableMask</code> (modal uniquement) ; désactivé par défaut pour éviter les fermetures accidentelles.</td>
    </tr>
    <tr>
      <td><strong>Motion</strong></td>
      <td>Respecte <code>prefers-reduced-motion</code> et l'interrupteur global <code>data-motion="off"</code> ; désactivable par dialogue via <code>motionDisabled</code>.</td>
    </tr>
  </tbody>
</table>
