# ui-textarea

Zone de texte **multiligne** bâtie sur le shell partagé
[`ui-field`](?path=/docs/components-ui-forms-ui-field--docs) (label + boîte + helper) en mode
**multiline**, avec un `<textarea>` natif. Couleurs via tokens `form.*`, bordure pilotée par le
`level`. Redimensionnement via la poignée **native** du navigateur (`resize`, verticale par défaut,
dans le coin bas-droit) ou `autoResize` (la hauteur suit le contenu). Compteur de caractères
optionnel (`showCount` + `maxlength`).

Autonome, `[(ngModel)]` ou reactive forms (`ControlValueAccessor` via `BaseFormField`). Pour une
saisie sur une seule ligne → [`ui-input`](?path=/docs/components-ui-forms-ui-input--docs).

## Niveaux & états

`level` (`default`/`success`/`error`) pilote la bordure. Le niveau `success` est un état **visuel
manuel** (ex. après une vérification). L'état d'**erreur** est en revanche piloté par la validation :
la bordure rouge et `errorText` apparaissent quand le contrôle est **invalide ET touché/modifié**
(règle `showError`), et disparaissent dès que la saisie redevient valide.

## Validation

Avec les formulaires réactifs (ou template-driven), l'erreur est **dynamique** : elle n'apparaît
qu'après interaction et se lève automatiquement quand le champ redevient valide. `required` affiche
aussi le marqueur `*` sur le label. Pas besoin de forcer `level="error"`.

<Canvas of={TextareaStories.Validation} />

## Redimensionnement

Par défaut, la poignée native permet d'agrandir **verticalement** le champ. `resize`
(`none`/`vertical`/`horizontal`/`both`) change l'axe. Avec `autoResize`, la hauteur suit le contenu
saisi et la poignée est désactivée.

```html
<!-- Reactive forms + message d'erreur -->
<ui-textarea formControlName="bio" label="Bio" errorText="Minimum 20 caractères" required />

<!-- Auto-resize : la hauteur suit le contenu -->
<ui-textarea [(ngModel)]="message" label="Message" autoResize />
```

> Architecture : `ui-textarea` partage le shell `ui-field` (label + boîte + helper + états) avec
> `ui-input` et la base `BaseFormField` (CVA + validation). Le mode `multiline` de `ui-field`
> ouvre la boîte (haut-alignée, hauteur automatique avec `min-height`). Le sous-composant Figma
> `_ui-textarea-content` est porté nativement par le `<textarea>`, et le « resizer » par la poignée
> native.

sont partagées dans le groupe [Forms](?path=/docs/components-configuration-forms--docs) ; voir aussi
le [fonctionnement de la configuration](?path=/docs/components-configuration--docs).
