# ui-input

Champ **texte** (text/email/tel/url/search/**password**) bâti sur le shell partagé
[`ui-field`](?path=/docs/components-ui-forms-ui-field--docs) (label + boîte + helper) et un
`<input>` natif. Couleurs via tokens `form.*`, bordure pilotée par le `level`. Slots
optionnels : icône gauche, unité suffixe, et **zone d'action droite**.

Autonome, `[(ngModel)]` ou reactive forms (`ControlValueAccessor` via `BaseFormField`).
Pour les nombres → [`ui-input-number`](?path=/docs/components-ui-forms-ui-input-number--docs) ;
pour les masques → [`ui-input-mask`](?path=/docs/components-ui-forms-ui-input-mask--docs).

## Niveaux & états

`level` (`default`/`success`/`error`) pilote la bordure. `errorText` remplace l'aide quand le
champ est en erreur (`invalid` explicite, ou automatique : contrôle invalide + touché).

## Slots & zone d'action

L'icône droite devient une **zone d'action** (bouton carré pleine hauteur, focusable) dès qu'on
renseigne `iconRightAriaLabel` ; le clic émet `iconRightClick`.

```html
<!-- Reactive forms + message d'erreur -->
<ui-input formControlName="email" type="email" label="Email" errorText="Adresse invalide" required />

<!-- Recherche : icône « effacer » actionnable, visible seulement avec du texte -->
<ui-input
  [(ngModel)]="q"
  label="Recherche"
  iconLeft="magnifying-glass"
  [iconRight]="q ? 'xmark' : undefined"
  iconRightAriaLabel="Effacer"
  (iconRightClick)="q = ''"
/>
```

> Architecture : `ui-input`, `ui-input-number` et `ui-input-mask` partagent le shell `ui-field`
> (label + boîte + helper + états) et la base `BaseFormField` (CVA + validation). Le
> sous-composant Figma `_ui-input-content` est porté nativement par l'`<input>`.
