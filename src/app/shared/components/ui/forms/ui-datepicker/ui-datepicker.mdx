# ui-datepicker

Sélecteur de **date / mois / année** (et d'**heure** optionnelle) headless. Un champ
déclencheur [`ui-input`](?path=/docs/components-ui-forms-ui-input--docs) en lecture seule ouvre
un calendrier stylé (tokens `form.*` / `actions.*`) dans un overlay Angular CDK — ou rendu
**inline**. Sélection `single` / `multiple` / `range`, vues à tiroir (jour → mois → année),
modes `MonthPicker` / `YearPicker`, plusieurs mois côte à côte, navigation mensuelle, focus
roving et support clavier complet (motif WAI-ARIA date picker).
`p-datepicker`.

Valeur `Date | null` câblée via `ControlValueAccessor` (`[(ngModel)]` ou reactive forms) sur
[`BaseFormField`](?path=/docs/components-ui-forms-ui-input--docs) : `label`, `helperText`/`errorText`,
`level`, la validation et les états `disabled`/`readonly` sont partagés avec les autres champs.

## API

## Sélection : single / multiple / range

`selectionMode` définit la quantité sélectionnée. La valeur du modèle est un `Date` en `single`,
un `Date[]` en `multiple` (bascule au clic, le panneau reste ouvert) et un `Date[]` `[début, fin]`
en `range` (1er clic = début, 2ᵉ = fin, surbrillance de l'intervalle).

<Canvas of={DatepickerStories.Range} />

## Vues & modes (jour → mois → année)

Un clic sur le **titre** de l'entête remonte d'un niveau : jour → sélection de mois → sélection
d'année. L'input `view` fixe la granularité de base : `date` (défaut), `month`
(**MonthPicker**) ou `year` (**YearPicker**). En mode mois/année, le clic sélectionne directement
un mois / une année.

<Canvas of={DatepickerStories.MonthPicker} />
<Canvas of={DatepickerStories.YearPicker} />

## Plusieurs mois

`numberOfMonths` affiche N panneaux mois côte à côte, séparés par un filet, avec les titres de
mois alignés en face des flèches ← → (aux extrémités). Le changement de vue à tiroir reste
réservé au mode mono-mois.

<Canvas of={DatepickerStories.TwoMonths} />

## Ouverture (placement)

Le panneau s'ouvre sous le champ et **se retourne automatiquement au-dessus** quand la place
manque en bas (CDK Overlay). `autoFlip="false"` verrouille l'ouverture vers le bas.

<Canvas of={DatepickerStories.SmartPosition} />

## Date & heure

`showTime` ajoute une ligne horaire sous forme de **steppers verticaux** (chevrons ↑/↓ autour de
chaque unité HH : MM — pattern ARIA `spinbutton`, pilotables aux flèches clavier) ;
`hourFormat="12"` ajoute un sélecteur AM/PM. `timeOnly` masque le calendrier. Quand `showTime` est
actif, le panneau **reste ouvert** après le choix d'un jour (pour régler l'heure) même si `closeOnSelect`.

<Canvas of={DatepickerStories.WithTime} />
<Canvas of={DatepickerStories.Time12h} />

## Contraintes de sélection

`minDate` / `maxDate` bornent la plage ; `disabledDays` (0 = dimanche … 6 = samedi) et
`disabledDates` (dates ponctuelles) neutralisent des cellules. `showButtonBar` ajoute les actions
« Aujourd'hui » / « Effacer ».

## Templates

Deux points d'extension via `<ng-template>` projetés, résolus par `contentChild` :

- **`#date`** — cellule de jour personnalisée. Contexte : `$implicit` = descripteur du jour
  (`{ day, month, year, otherMonth, today, selected, disabled, rangeStart, rangeEnd, inRange }`)
  et `selected` (booléen). Idéal pour des pastilles d'évènements, week-ends colorés, etc.
- **`#buttonbar`** — barre de boutons personnalisée. Contexte : `todayCallback` et `clearCallback`
  (fonctions à câbler sur `(click)`).

<Canvas of={DatepickerStories.DateTemplate} />
<Canvas of={DatepickerStories.CustomButtonBar} />

```html
<!-- Cellule de jour personnalisée -->
<ui-datepicker [(ngModel)]="date">
  <ng-template #date let-d let-selected="selected">
    <span>{{ d.day }}</span>
    @if (hasEvent(d)) { <span class="dot"></span> }
  </ng-template>
</ui-datepicker>

<!-- Barre de boutons personnalisée -->
<ui-datepicker [(ngModel)]="range" selectionMode="range" showButtonBar>
  <ng-template #buttonbar let-todayCallback="todayCallback" let-clearCallback="clearCallback">
    <ui-button size="small" label="Aujourd'hui" (buttonClick)="todayCallback($event)" />
    <ui-button size="small" level="error" icon="xmark" iconOnly ariaLabel="Effacer" (buttonClick)="clearCallback($event)" />
  </ng-template>
</ui-datepicker>
```

```html
<!-- Reactive forms -->
<ui-datepicker formControlName="birth" label="Date de naissance" [maxDate]="today" required />

<!-- Date + heure, ngModel -->
<ui-datepicker [(ngModel)]="appointment" label="Rendez-vous" showTime hourFormat="12" showButtonBar />

<!-- Inline (pas d'overlay) -->
<ui-datepicker [(ngModel)]="date" inline />
```

> **Périmètre / choix** : le champ déclencheur est en **lecture seule** (choix par le calendrier ;
> pas de saisie/parsing texte libre). Le tiroir de vues (jour → mois → année) et le focus roving
> clavier sont actifs en **mono-mois** ; en multi-mois les entêtes sont fixes avec flèches aux
> extrémités. Les états interactifs (hover/focus/pressed) sont des **pseudo-classes CSS** pilotées
> par les tokens, pas des props. Personnalisation via `panelStyleClass`, `#date`, `#buttonbar`.
