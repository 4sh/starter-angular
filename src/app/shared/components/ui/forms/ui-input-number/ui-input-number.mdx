# ui-input-number

Champ **numérique** bâti sur le shell [`ui-field`](?path=/docs/components-ui-forms-ui-field--docs).
Valeur modèle = `number | null`. Saisie tolérante (pas de reformatage pendant la frappe →
pas de saut de curseur), `min`/`max`/`step`, spinner ± et flèches clavier ↑/↓, **clamp au blur**.

Autonome, `[(ngModel)]` ou reactive forms.

## Formatage (Intl.NumberFormat)

Le format riche (groupement des milliers, devise, décimales) est appliqué **au blur** ; au
focus, le champ bascule sur une forme éditable (sans groupement/devise) pour une saisie fluide.
Piloté par `locale` / `currency` / `useGrouping` / `minFractionDigits` / `maxFractionDigits`.

```html
<ui-input-number formControlName="age" label="Âge" [min]="0" [max]="120" [allowDecimals]="false" />
<ui-input-number [(ngModel)]="price" label="Prix" locale="fr-FR" currency="EUR" [step]="0.5" />
```

> Cas très spécifique : surcharger la méthode protégée `format(value: number): string` dans une
> sous-classe (le parsing locale-aware reste géré par le composant).

## Theming

Aucune variable locale surchargeable : la boîte vient du shell `ui-field`. La configuration passe
par ses variables (groupe partagé [Forms](?path=/docs/components-configuration-forms--docs)),
détaillées sur la doc [`ui-input`](?path=/docs/components-ui-forms-ui-input--docs). Voir le
[fonctionnement de la configuration](?path=/docs/components-configuration--docs).
