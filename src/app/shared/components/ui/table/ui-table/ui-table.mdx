# ui-table

Tableau de données **headless** : le composant orchestre le pipeline de données
(tri → pagination), la sélection, l'expansion, le scroll et la pagination, le
**markup des lignes reste entièrement à la main du consommateur** via les
templates `#header` / `#body` / `#footer` (de vrais `<tr>/<th>/<td>`). Les
comportements de colonne s'activent à la carte avec les directives compagnes
(`uiSortableColumn`, `uiSelectableRow`, `uiFrozenColumn`, `uiResizableColumn`,
`uiRowToggler`) et les sous-composants (`ui-table-sort-icon`,
`ui-table-checkbox`, `ui-table-header-checkbox`, `ui-table-radio`).

## Basic

Une collection à afficher et un template de colonnes suffisent.

<Canvas of={TableStories.Basic} />

```html
<ui-table [value]="products">
  <ng-template #header>
    <tr>
      <th>Code</th>
      <th>Nom</th>
    </tr>
  </ng-template>
  <ng-template #body let-product>
    <tr>
      <td>{{ product.code }}</td>
      <td>{{ product.name }}</td>
    </tr>
  </ng-template>
</ui-table>
```

## Selection

### Single

Une ligne à la fois : cliquer une autre ligne remplace la sélection. Poser
`[uiSelectableRow]` sur le `<tr>` du template body.

<Canvas of={TableStories.SelectionSingle} />

### Multiple

Plusieurs lignes sans colonne dédiée. Avec `metaKeySelection`, Ctrl/Cmd +
clic bascule une ligne, Maj + clic sélectionne une plage, un clic simple
remplace la sélection.

<Canvas of={TableStories.SelectionMultiple} />

### Checkbox

Sélection multiple par cases à cocher : `ui-table-header-checkbox` en
en-tête (tout sélectionner, indéterminée sur sélection partielle) et
`ui-table-checkbox` sur chaque ligne.

<Canvas of={TableStories.SelectionCheckbox} />

### Radio

Sélection simple par boutons radio : `selectionMode="single"` +
`ui-table-radio` dans une colonne.

<Canvas of={TableStories.SelectionRadio} />

### Clavier

Sur les lignes `uiSelectableRow` : `↑`/`↓` déplacent le focus de ligne en
ligne, `Espace`/`Entrée` basculent la ligne focalisée, `Maj + flèche` étend
la plage. Combiné à `selectionMode="multiple"` + `metaKeySelection`, tout le
flux est pilotable au clavier.

### Column

La sélection depuis un élément d'une colonne (lien, bouton…) s'implémente
par templating : appelez vos propres handlers dans le template `#body`, la
sélection reste pilotable via le two-way `[(selection)]`.

### Events

`rowSelect` et `rowUnselect` notifient chaque (dé)sélection avec
`{ originalEvent, data, index, type }`.

<Canvas of={TableStories.SelectionEvents} />

## Sort

### Single

`uiSortableColumn` + `ui-table-sort-icon` sur un `<th>` : chaque clic (ou
`Entrée`/`Espace`) cycle croissant → décroissant → non trié.

<Canvas of={TableStories.SortSingle} />

```html
<th uiSortableColumn="name">Nom <ui-table-sort-icon field="name" /></th>
```

### Multiple

`sortMode="multiple"` : Ctrl/Cmd + clic compose le tri sur plusieurs
colonnes (un clic simple repart sur la colonne seule).

<Canvas of={TableStories.SortMultiple} />

### Presort

`sortField` + `sortOrder` appliquent un tri au montage ; les en-têtes
restent interactifs (`multiSortMeta` pour le mode multiple).

<Canvas of={TableStories.SortPresort} />

### Removable / custom

Le cycle simple retire déjà le tri au troisième clic. Pour un ordre
entièrement personnalisé, `customSort` désactive le tri interne : la table
émet `sortFunction` avec `{ data, mode, field, order, multiSortMeta }` et
c'est au consommateur d'ordonner `value`.

## Pagination

### Basic

`paginator` + `rows` découpent la collection ; `rowsPerPageOptions` ajoute
le sélecteur de lignes par page. La barre est une instance du composant
autonome **`ui-paginator`** (voir sa doc dédiée), composée de `ui-button`
(contrôles circulaires, page courante en `high`) et `ui-select`.

<Canvas of={TableStories.PaginationBasic} />

### Programmatic

Le modèle two-way `first` pilote la pagination depuis l'extérieur, tout
changement déclenche la re-pagination et `pageChange`.

<Canvas of={TableStories.PaginationProgrammatic} />

## Scroll

### Vertical

`scrollable` + `scrollHeight` fixe activent le scroll vertical avec en-tête
sticky (et pied sticky).

<Canvas of={TableStories.ScrollVertical} />

### Horizontal

Quand la somme des largeurs dépasse le conteneur, la table défile
horizontalement, donnez un `min-width` à chaque colonne pour éviter
l'écrasement (via `tableStyle` + styles de colonne).

<Canvas of={TableStories.ScrollHorizontal} />

### Flexible

`scrollHeight="flex"` laisse le viewport grandir et rétrécir avec son parent
flex, pratique dans une modale redimensionnable ou un split layout.

<Canvas of={TableStories.ScrollFlexible} />

## Frozen Columns

`uiFrozenColumn` fige une colonne pendant le scroll horizontal ;
`alignFrozen` la colle à gauche (défaut) ou à droite. Plusieurs colonnes
peuvent être figées de chaque côté, les décalages s'accumulent
automatiquement.

<Canvas of={TableStories.FrozenColumns} />

## Frozen Rows

`frozenValue` épingle des lignes sous l'en-tête pendant le scroll vertical.

<Canvas of={TableStories.FrozenRows} />

## Row Expansion

`uiRowToggler` sur le bouton d'expansion + template `#expandedrow` pour le
contenu déplié. L'état vit dans le two-way `expandedRowKeys` (nécessite
`dataKey`) ; `rowExpand`/`rowCollapse` notifient chaque bascule.

<Canvas of={TableStories.RowExpansion} />

## Column Resize (fit)

`resizableColumns` + `uiResizableColumn` sur les `<th>` : le glissement
prend la largeur sur la colonne voisine, la largeur totale de la table ne
change pas. `showGridlines` rend l'effet visible.

<Canvas of={TableStories.ColumnResizeFit} />

## Row Reorder

`uiReorderableRow` sur le `<tr>` (avec le `rowIndex` du contexte) rend les
lignes réordonnables par glisser-déposer ; un indicateur marque le bord de
dépose. `rowReorder` émet `{ dragIndex, dropIndex, value }` avec la **copie
réordonnée de `value`** à re-binder. Prévu pour des données non triées.

<Canvas of={TableStories.RowReorder} />

## Column Toggle

L'affichage dynamique des colonnes est du pur templating : pilotez la liste
des colonnes avec un `ui-select multiple` et bouclez dessus dans `#header`
et `#body`.

<Canvas of={TableStories.ColumnToggle} />

## Loading

Pendant le chargement (`loading`), les lignes du corps sont **remplacées par
une ligne d'attente** (`ui-spinner` centré) — l'en-tête reste net et en
place, sans voile translucide par-dessus. Les lignes apparaissent une fois
les données chargées ; la racine passe `aria-busy="true"` et le spinner est
une région `role="status"` nommée.

<Canvas of={TableStories.Loading} />

## Skeleton

Pendant qu'une requête est en vol, on peut rendre des **lignes de substitution**
remplies de `ui-skeleton` plutôt qu'un voile de chargement. On alimente le
tableau avec un tableau d'espaces réservés et le template `#body` place un
`ui-skeleton` par cellule : la structure (en-têtes, colonnes, densité) reste
identique au contenu réel, ce qui évite tout saut de mise en page à l'arrivée
des données.

<Canvas of={TableStories.Skeleton} />

## Virtual Scroll

### Preload

`virtualScroll` + `virtualScrollItemSize` (hauteur fixe, à reporter sur le
`<tr>` du template body) fenêtrent le rendu sur le viewport visible,
10 000 lignes préchargées se rendent en quelques dizaines de `<tr>`.
Requiert `scrollable` + un `scrollHeight` fixe.

<Canvas of={TableStories.VirtualScrollPreload} />

### Lazy

`lazy` + l'output `lazyLoad { first, last }` notifient chaque déplacement de
la fenêtre rendue : chargez la plage à la demande et re-bindez `value`.

<Canvas of={TableStories.VirtualScrollLazy} />

## États

- **Ligne survolée** : `table.body.surface.hover` (dès `rowHover` ou un mode de sélection).
- **Ligne sélectionnée** : famille `informative.highlightLow` (la catégorie `table.*` n'a pas d'état `selected`).
- **Ligne paire** (striped) : `table.body.*.even`.
- **En-tête triable survolé** : `table.head.surface.hover` ; focus visible en anneau intérieur `global.high.stroke.focused`.
- Les états interactifs sont des **pseudo-classes CSS**, jamais des props.
