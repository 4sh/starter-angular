# ui-alert

Message **inline** pour un retour informatif, de succès, d'avertissement ou d'erreur.

Contrairement à [`ui-toast`](/docs/components-ui-informative-ui-toast--docs) (notification
flottante, pilotée par service), `ui-alert` s'insère dans le flux de la page : bandeau de
formulaire, résumé d'erreurs, statut inline. La racine porte `role="alert"`
(`aria-live="assertive"` + `aria-atomic="true"` implicites) : le message est annoncé dès
son apparition.

## Icon

L'icône de tête se pilote via `icon` : un nom FontAwesome pour surcharger,
`false` pour la masquer, `true` (défaut) pour l'icône par défaut du niveau.

<Canvas of={AlertStories.CustomIcon} />
<Canvas of={AlertStories.NoIcon} />

## Variant

Le sous-niveau (`subLevel`) module l'intensité : `high` (soutenu) ou `low` (discret).

<Canvas of={AlertStories.SubLevelLow} />

## Sizes

Deux tailles : `default` et `large`.

<Canvas of={AlertStories.Large} />

## Dynamic

Plusieurs messages s'affichent via un bloc `@for` ; chacun se retire à la fermeture.

<Canvas of={AlertStories.Dynamic} />

## Life

Renseigner `life` (ms) fait disparaître l'alerte automatiquement (elle se masque et
émet `close`).

<Canvas of={AlertStories.Life} />

```html
<ui-alert level="success" text="Enregistré." [life]="4000" (close)="onClose()" />
```

## Forms

Les erreurs de validation d'un formulaire s'affichent en sévérité `error`.

<Canvas of={AlertStories.Forms} />
