# ui-tooltip

Directive `[uiTooltip]` fournissant une **infobulle** (advisory) sur n'importe quel élément
déclencheur. Le comportement est **headless** et s'appuie sur **Angular CDK Overlay** : le
placement, le **retournement automatique** quand l'espace manque dans le viewport, et le
repositionnement au scroll sont délégués au `FlexibleConnectedPositionStrategy` (aucune
géométrie manuelle). Adaptée aux signals.

Le panneau visuel (bulle + flèche) est **100 % piloté par les design tokens** de la catégorie
`informative` (`--informative-defaulthigh-*`). Personnalisation ponctuelle via `tooltipStyleClass`
et contenu riche via `TemplateRef`.

## API

<ArgTypes of={TooltipStories} />

> Le contenu se lie via l'alias de la directive : `[uiTooltip]="…"`. Il accepte une chaîne,
> du HTML (avec `escape=false`, assaini par Angular) ou un `TemplateRef` (contexte via
> `[tooltipContext]`).

## Placement

Quatre côtés d'ancrage (`top` par défaut). Par défaut (`fitContent=true`), le CDK bascule vers
un côté de repli si l'infobulle sortirait du viewport, et la flèche suit le côté effectif.
Passez `fitContent=false` pour **verrouiller** le côté choisi (aucun repli).

> **`appendTo`** : le CDK attache l'overlay à un conteneur au niveau `<body>`. C'est volontaire : cela évite le rognage par un ancêtre
> `overflow`/`transform` et les conflits de `z-index`. Il n'y a donc pas de prop `appendTo`.

## Interactivité & personnalisation

`autoHide=false` garde l'infobulle ouverte tant qu'on la survole (contenu sélectionnable /
cliquable). Pour un rendu ponctuel différent, posez `tooltipStyleClass` sur le panneau et
ciblez cette classe depuis une feuille de style globale.

## Bonnes pratiques

- **Déclencheur natif / focusable** : posez la directive sur un `<button>`, un `ui-button`,
  un lien ou un champ. `tooltipEvent="both"` (défaut) affiche aussi au **focus clavier**.
- **Concision** : une infobulle porte une aide brève, jamais une information essentielle ni
  un contenu interactif indispensable (utilisez un popover/dialog pour cela).
- **Délais** : `showDelay` (150 ms) évite le clignotement au survol de passage ; `hideDelay`
  laisse le temps d'atteindre une infobulle interactive.

## Exemples

```html
<!-- Texte simple -->
<button uiTooltip="Enregistrer le document">Enregistrer</button>

<!-- Côté + délais -->
<ui-button label="Aide" uiTooltip="Détails…" tooltipPosition="right" [showDelay]="300" />

<!-- Contenu riche (TemplateRef) -->
<button [uiTooltip]="tip" [tooltipContext]="{ $implicit: user }" [autoHide]="false">Profil</button>
<ng-template #tip let-u><strong>{{ u.name }}</strong><br />{{ u.role }}</ng-template>

<!-- HTML brut (assaini) -->
<span [uiTooltip]="'<b>Gras</b> autorisé'" [escape]="false">Info</span>
```

## Theming

Le panneau consomme **directement** les design tokens de la catégorie `informative`
(comme `ui-badge`) — surface / contenu / bordure via `--informative-defaulthigh-*`, rayon
`--radius-default`, rembourrage `--units-*`, ombre `--shadow-default-md`. Aucun hook CSS
spécifique au composant : basculer un thème/mode suffit à re-teinter l'infobulle.

Deux constantes structurelles restent **locales** (`ui-tooltip.scss`), propres à ce composant :

**Rendu ponctuel** : `tooltipStyleClass` ajoute une classe sur le panneau ; ciblez-la depuis une
feuille de style **globale**. Le panneau utilisant des styles scopés, le plus fiable est de
**re-mapper les tokens sémantiques** sur cette classe plutôt que d'écraser les propriétés
(la substitution de variable évite tout conflit de spécificité, et la flèche suit) :

```css
.tip-highlight {
  --informative-defaulthigh-surface-default: var(--informative-highlighthigh-surface-default);
  --informative-defaulthigh-content-default: var(--informative-highlighthigh-content-default);
  --informative-defaulthigh-stroke-default:  var(--informative-highlighthigh-stroke-default);
}
```

Pour des variantes sémantiques récurrentes (`highlight`, `success`, …), la voie idiomatique
serait d'ajouter un input `level` générant des modifiers `._{level}` façon `ui-badge` (non
implémenté ici, hors périmètre Figma).
Voir aussi [Components/Configuration](?path=/docs/components-configuration--docs).
