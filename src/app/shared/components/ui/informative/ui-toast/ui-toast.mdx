# ui-toast

Notification transitoire, empilée dans un coin de l'écran. Le composant se décline en
trois pièces complémentaires :

- **`UiToast`** : la carte présentational (icône · titre/texte · fermeture). Entièrement
  pilotée par les tokens `informative` (surface / content / stroke), en `light` **et** `dark`.
- **`UiToastService`** : le point d'entrée programmatique (`add` / `addAll` / `clear` / `remove`).
- **`ui-toast-container`** : la pile ancrée dans le viewport : elle lit le service, positionne
  les toasts, anime l'entrée/sortie via le **système de motion** partagé et gère l'auto-fermeture
  (mise en pause au survol / focus).

> **`breakpoints` (PrimeNG)** n'est **pas** repris : la largeur de la pile est déjà responsive
> (bornée au viewport via `min(largeur, calc(100vw - marges))`). Pour des styles par palier,
> passer une classe via `styleClass` et cibler des `@media` en CSS : plutôt que des styles inline
> par breakpoint, non alignés avec l'approche token-first du design system.

## Empilement (limite, gap, doublons)

`stackVisibleLimit` plafonne le nombre de toasts visibles, `stackGap` resserre l'espacement,
et `preventDuplicates` fusionne les messages identiques.

<Canvas of={ToastStories.Stacking} />

## Motion

L'entrée/sortie est câblée sur le **système de motion** partagé (`animate.enter` /
`animate.leave` + directive `uiMotion`). Le preset est choisi selon la position
(`slide-down` en haut, `slide-up` en bas, `zoom` au centre) et surchargeable via l'input
`motion`. Le kill-switch global `data-motion="off"` et la préférence *reduced-motion* sont
respectés automatiquement.

## Accessibilité

<table className="doc-table">
  <thead>
    <tr><th>Point</th><th>Implémentation</th></tr>
  </thead>
  <tbody>
    <tr>
      <td><strong>Région live</strong></td>
      <td>Chaque toast est annoncé : <code>role="alert"</code> + <code>aria-live="assertive"</code> pour <code>error</code>/<code>warning</code>, <code>role="status"</code> + <code>aria-live="polite"</code> sinon. <code>aria-atomic="true"</code>.</td>
    </tr>
    <tr>
      <td><strong>Fermeture</strong></td>
      <td><code>&lt;button&gt;</code> natif avec <code>aria-label</code> (« Fermer »), focusable au clavier, anneau de focus <code>:focus-visible</code> distinct.</td>
    </tr>
    <tr>
      <td><strong>Pause au survol / focus</strong></td>
      <td>Le compte à rebours d'auto-fermeture se met en pause au survol et au focus clavier : l'utilisateur a le temps de lire et d'agir (WCAG « enough time »).</td>
    </tr>
    <tr>
      <td><strong>Icône</strong></td>
      <td>Décorative (<code>aria-hidden</code> via <code>ui-icon</code>) ; la couleur n'est jamais le seul vecteur de sens (icône + titre explicites).</td>
    </tr>
    <tr>
      <td><strong>Sticky pour les actions</strong></td>
      <td>Un toast porteur d'actions doit être <code>sticky</code> : il ne disparaît pas pendant que l'utilisateur s'apprête à cliquer.</td>
    </tr>
  </tbody>
</table>
