# ui-button-split

Un **bouton d'action** principal accolé à un **déclencheur déroulant** qui révèle
une collection d'options additionnelles. La logique de composition réutilise
[`ui-button`](?path=/docs/components-ui-actions-ui-button--docs) (les deux boutons,
tokens `actions.*`) et [`ui-menu`](?path=/docs/components-ui-navigation-ui-menu--docs)
(le panneau, tokens `navigation.*`). Les options sont pilotées par un modèle
déclaratif `UiMenuItem[]` — le même que celui du menu.

## API

<ArgTypes of={ButtonSplitStories} />

## Basic

Le bouton d'action déclenche l'action par défaut (`buttonClick`) ; le déclencheur
ouvre un menu d'options additionnelles défini par la propriété `model`, basée sur
le modèle `UiMenuItem`.

```html
<ui-button-split label="Enregistrer" [model]="options" (buttonClick)="save()" (itemClick)="onOption($event)" />
```

## Icons

Les boutons comme les options du menu supportent l'affichage d'icônes.

```html
<ui-button-split label="Enregistrer" icon="floppy-disk" [model]="optionsWithIcons" />
```

<Canvas of={ButtonSplitStories.Icons} />

## Nested

Les options peuvent s'imbriquer en sous-menus (groupes repliables), toujours via le
modèle `UiMenuItem` (un item avec `items` devient un groupe).

<Canvas of={ButtonSplitStories.Nested} />

## Level

La propriété `level` définit le type de bouton.

<Canvas of={ButtonSplitStories.Level} />

## Disabled

Avec l'attribut `disabled`, le composant est inactif et non focusable. Les états
désactivés du bouton et du déclencheur se gèrent aussi indépendamment : le bouton
d'action est désactivé via `buttonDisabled`, le déclencheur du menu via
`menuButtonDisabled`.
