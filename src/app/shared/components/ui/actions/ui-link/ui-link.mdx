# ui-link

Lien **textuel inline** : un vrai `<a>`, destiné au fil du texte (paragraphes) et à la
navigation. Stylé avec la famille sobre `actions.low.content.*` (tokens light **et** dark).

> **Choisir par la sémantique, pas par l'apparence.**
> - Ce qui **navigue** (URL / route) → `ui-link`, ou `ui-button` avec `href`/`routerLink`
>   quand il faut l'apparence d'un bouton.
> - Ce qui **déclenche une action** (submit, ouvrir un modal…) → `ui-button`.

Accepte une destination externe (`href`) **ou** interne (`routerLink`, directive Angular).

## Lien externe

`external` force `target="_blank"` et pose un `rel="noopener noreferrer"` sûr.

<Canvas of={LinkStories.External} />

## Navigation interne (RouterLink)

`routerLink` bascule sur la directive Angular pour la navigation SPA (le comportement
natif de l'ancre — Cmd/Ctrl+clic, etc. — est préservé).

<Canvas of={LinkStories.RouterInternal} />

```html
<ui-link label="Mon profil" [routerLink]="['/profil']" iconRight="chevron-right" />
```

## Accessibilité

    <tr>

  <tbody>
    <tr>
      <td><strong>Sémantique</strong></td>
      <td><code>&lt;a&gt;</code> natif. Un lien navigue ; pour une action, utiliser <code>ui-button</code>.</td>
    </tr>
    <tr>
      <td><strong>Soulignement</strong></td>
      <td>Souligné par défaut : la couleur n'est jamais le seul repère d'un lien inline. Retiré en mode <em>icon-only</em>.</td>
    </tr>
    <tr>
      <td><strong>Labeling</strong></td>
      <td><code>aria-label</code> posé quand utile (icon-only). Un warning dev signale un lien icon-only sans nom accessible.</td>
    </tr>
    <tr>
      <td><strong>Externe</strong></td>
      <td><code>external</code> / <code>target="_blank"</code> ⇒ <code>rel="noopener noreferrer"</code> automatique.</td>
    </tr>
    <tr>
      <td><strong>États</strong></td>
      <td><code>:focus-visible</code> visible et distinct du <code>hover</code>. <code>disabled</code> : href/routerLink retirés, <code>aria-disabled</code>, hors tabulation (<code>tabindex=-1</code>).</td>
    </tr>
    <tr>
      <td><strong>Contexte</strong></td>
